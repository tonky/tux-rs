# 2.2 — Trait Hierarchy + Mock Backend

**Phase:** 2 — tux-core: Hardware Model System
**Estimate:** 1 day (6 hours)
**Depends on:** 2.1
**Unlocks:** 4.1, 4.2 (fan backends use these traits)

## Goal

Define the hardware abstraction traits that platform backends implement, plus mock implementations for testing. These traits are the contract between tux-core types and tux-daemon platform-specific code.

### Traits

```rust
// backend/fan.rs
pub trait FanBackend: Send + Sync {
    fn read_temp(&self) -> io::Result<u8>;                          // °C
    fn write_pwm(&self, fan_index: u8, pwm: u8) -> io::Result<()>; // 0–255
    fn read_pwm(&self, fan_index: u8) -> io::Result<u8>;
    fn set_auto(&self, fan_index: u8) -> io::Result<()>;
    fn read_fan_rpm(&self, fan_index: u8) -> io::Result<u16>;
    fn num_fans(&self) -> u8;
}

// backend/sensor.rs
pub trait SensorBackend: Send + Sync {
    fn read_temperatures(&self) -> io::Result<Vec<(String, f32)>>;  // label → °C
    fn read_fan_rpms(&self) -> io::Result<Vec<(String, u16)>>;      // label → RPM
}

// backend/keyboard.rs
pub trait KeyboardBackend: Send + Sync {
    fn set_brightness(&self, brightness: u8) -> io::Result<()>;
    fn set_color(&self, zone: u8, r: u8, g: u8, b: u8) -> io::Result<()>;
    fn set_mode(&self, mode: &str) -> io::Result<()>;
    fn zone_count(&self) -> u8;
    fn available_modes(&self) -> Vec<String>;
}

// backend/profile.rs
pub trait ProfileBackend: Send + Sync {
    fn get_profile(&self) -> io::Result<String>;
    fn set_profile(&self, profile: &str) -> io::Result<()>;
    fn available_profiles(&self) -> Vec<String>;
}
```

### Mock Implementations

```rust
// mock/fan.rs
pub struct MockFanBackend {
    temp: AtomicU8,
    pwm: Vec<AtomicU8>,     // per-fan
    rpm: Vec<AtomicU16>,
    auto_mode: Vec<AtomicBool>,
    num_fans: u8,
}

impl MockFanBackend {
    pub fn new(num_fans: u8) -> Self;
    pub fn set_temp(&self, temp: u8);
```

### Mock Sysfs Helpers

```rust
// mock/sysfs.rs
pub struct MockSysfs {
    root: TempDir,

impl MockSysfs {
```
