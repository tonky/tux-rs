# 2.3 — Device Table (40+ SKUs)

**Phase:** 2 — tux-core: Hardware Model System
**Estimate:** 1 day (6 hours)
**Depends on:** 2.1
**Unlocks:** 2.4 (detection looks up table), Phase 4 (backends use descriptors)

## Goal

Populate the static device table with all known TUXEDO laptop models. Each entry maps a DMI product SKU to a full DeviceDescriptor with capabilities and register addresses. Adding a new laptop model = adding a table entry.

## Deliverables

### Module Structure

```
tux-core/src/
├── device_table.rs      # Static DEVICE_TABLE + lookup functions
└── ...
```

### Device Table

```rust
pub static DEVICE_TABLE: &[DeviceDescriptor] = &[
    // --- NB05 Platform ---
    DeviceDescriptor {
        name: "TUXEDO Pulse 14 Gen3",
        product_sku: "PULSE1403",
        platform: Platform::Nb05,
        fans: FanCapability { count: 2, control: FanControlType::Pwm { scale_max: 184 } },
        keyboard: KeyboardType::White,
        sensors: SensorSet { cpu_temp: true, gpu_temp: false, fan_rpm: true },
        charging: ChargingCapability::None,
        tdp: None,
        registers: PlatformRegisters::Nb05(Nb05Registers {
            num_fans: 2,
            fanctl_onereg: false,
            // ... register addresses
        }),
    },
    // ... 40+ more entries
];
```

### Lookup Functions

```rust
/// Find device by exact DMI product SKU match.
pub fn lookup_by_sku(sku: &str) -> Option<&'static DeviceDescriptor>;

/// Find all devices for a given platform.
pub fn devices_for_platform(platform: Platform) -> Vec<&'static DeviceDescriptor>;

/// Get the default/fallback descriptor for a platform (generic capabilities).
pub fn fallback_for_platform(platform: Platform) -> &'static DeviceDescriptor;
```

### SKU Sources

Populate from three sources:
1. **tuxedo-drivers-rs** `dmi.rs` — already has NB05 variants (Pulse 14 Gen3/Gen4, InfinityFlex 14)
2. **tuxedo-drivers** C source — `tuxedo_compatibility_check.h` has the full SKU list
3. **OVERVIEW.md** hardware table — capability matrix per platform

### Models to Include (minimum)

| Platform | Models |
|----------|--------|
| NB05 | Pulse 14 Gen3/Gen4, Pulse 15 Gen2, InfinityFlex 14 |
| Uniwill | InfinityBook Pro 14/15 Gen7/8, InfinityBook S 14/15 Gen8 |
| Clevo | Polaris 15/17 Gen5, Stellaris 15/17 Gen4/5, Stellaris Slim 15 |
| NB04 | Sirius 16 Gen1/Gen2 |
| Tuxi | Aura 15 Gen1/Gen2 |

### Fallback Descriptors

Each platform gets a generic fallback with conservative capabilities (e.g., 1 fan, no keyboard control) used when a specific SKU isn't matched but the platform is detected.

## Tests

- [ ] All SKUs in table are unique (no duplicates)
- [ ] Every Platform variant has at least one device entry
- [ ] `lookup_by_sku` returns correct device for known SKUs
- [ ] `lookup_by_sku` returns None for unknown SKU
- [ ] `devices_for_platform` returns correct subset
- [ ] `fallback_for_platform` returns valid descriptor for each platform
- [ ] All DeviceDescriptor entries have non-empty name and product_sku
- [ ] Table has ≥40 entries (completeness check)
