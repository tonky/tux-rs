### tuxedo-uw-fan вЂ” Uniwill EC Access

**Sysfs attributes** at `/sys/devices/platform/tuxedo-uw-fan/`:

| Attribute | Mode | Type | Description |
|-----------|------|------|-------------|
| `fan0_pwm` | RW | u8 | Fan 1 duty (0вЂ“200 EC scale) |
| `fan1_pwm` | RW | u8 | Fan 2 duty (0вЂ“200 EC scale) |
| `fan_mode` | RW | u8 | 0=auto, 1=manual |
| `cpu_temp` | RO | u8 | CPU temperature В°C |
| `gpu_temp` | RO | u8 | GPU temperature В°C |
| `fan_count` | RO | u8 | Always 2 |

**Hardware access:**
- ACPI methods `ECRR`/`ECRW` on `\_SB.INOU` (primary)
- Fallback EC paths if INOU not available
- EC registers: PWM @ 0x1804/0x1809, temp @ 0x043e/0x044f, manual mode @ 0x0751 bit 7
- 6ms delay between read/write operations (usleep_range)

### tuxedo-tuxi вЂ” ACPI TFAN

**Sysfs attributes** at `/sys/devices/platform/tuxedo-tuxi/`:

| Attribute | Mode | Type | Description |
|-----------|------|------|-------------|
| `fan0_pwm` | RW | u8 | Fan 1 speed (0вЂ“255 native) |
| `fan1_pwm` | RW | u8 | Fan 2 speed (0вЂ“255 native) |
| `fan_mode` | RW | u8 | 0=auto, 1=manual |
| `fan0_temp` | RO | u32 | Fan 1 temp (tenth-Kelvin from firmware) |
| `fan1_temp` | RO | u32 | Fan 2 temp (tenth-Kelvin from firmware) |
| `fan0_rpm` | RO | u16 | Fan 1 RPM |
| `fan1_rpm` | RO | u16 | Fan 2 RPM |
| `fan_count` | RO | u8 | Number of fans |

**Hardware access:**
- ACPI methods: GCNT (count), GSPD/SSPD (speed), GMOD/SMOD (mode), GTMP (temp), GRPM (RPM)
- Daemon converts tenth-Kelvin в†’ В°C: `(tenth_k - 2730) / 10`

### Design Rules (both modules)

- **Stateless passthrough** вЂ” no fan curves, no LED logic, no profiles
- Each read/write is a single ACPI method call or EC register access
- Mutex protection for concurrent access
- Standard platform_driver pattern: probe, remove, module_init/exit
- MODULE_LICENSE("GPL"), MODULE_AUTHOR, MODULE_DESCRIPTION
