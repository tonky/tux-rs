### tuxedo-ec вЂ” NB05 EC Port I/O

**Sysfs:** `/sys/devices/platform/tuxedo-ec/ec_ram` вЂ” 64 KiB binary attribute

**Access method:**
- SuperIO indirect I/O registers at port 0x4e (address) and 0x4f (data)
- I2EC protocol: 12 outb + 1 inb per read
- Single mutex protecting I2EC transactions
- Daemon uses `pread`/`pwrite` at offset = EC address (0x0000вЂ“0xFFFF)

**Key design:**
- No semantic attributes вЂ” raw EC memory exposed as a binary blob
- All interpretation (fan duty, temp, RPM register addresses) is in userspace
- This keeps the kernel module maximally simple and hardware-agnostic

### tuxedo-clevo вЂ” WMI/ACPI DSM Dual-Transport

**Sysfs attributes** at `/sys/devices/platform/tuxedo-clevo/`:

| Attribute | Mode | Type | Description |
|-----------|------|------|-------------|
| `fan0_info` | RO | u32 | Fan 1: bits[7:0]=duty, [15:8]=tempВ°C, [31:16]=RPM |
| `fan1_info` | RO | u32 | Fan 2: same encoding |
| `fan2_info` | RO | u32 | Fan 3: same encoding (if present) |
| `fan_speed` | WO | u32 | Packed: 3 fan duties in one write |
| `fan_auto` | WO | trigger | Restore hardware auto mode |

**Hardware access:**
- Primary: WMI (GUID `ABBC0F6D-...-...`) with event GUID `ABBC0F6B`
- Fallback: ACPI DSM (device HID `CLV0001`)
- Probe: issues cmd 0x52 (GET_BIOS_FEATURES_1), validates result в‰  0xFFFFFFFF
- Commands: 0x63/0x64/0x6e (FANINFO 1/2/3), 0x68 (SET_FANSPEED), 0x69 (SET_FANAUTO)
- 100ms delay after speed write (firmware processing time)

**FANINFO u32 parsing (done in userspace):**
```
duty = info & 0xFF              // 0вЂ“255
temp = (info >> 8) & 0xFF      // В°C
rpm  = (info >> 16) & 0xFFFF   // RPM
```
