# 3.3 — NB04 Shim + DKMS Packaging

**Phase:** 3 — tux-kmod: Kernel Shims
**Estimate:** 1 day (6 hours)
**Depends on:** 3.1, 3.2
**Unlocks:** Phase 4 (all kernel shims ready)

## Goal

Port the NB04 WMI kernel shim and create unified DKMS packaging for all 5 modules. After this sub-phase, `dkms build && dkms install` produces all shims ready for the daemon.

## Deliverables

### NB04 Module

```
tux-kmod/
├── tuxedo-nb04/
│   ├── tuxedo_nb04.c      # ~220 LOC
│   ├── Makefile
│   ├── Kbuild
│   └── dkms.conf
└── ...
```

**Sysfs attributes** at `/sys/devices/platform/tuxedo-nb04/`:

| Attribute | Mode | Type | Description |
|-----------|------|------|-------------|
| `cpu_temp` | RO | u8 | CPU temperature °C |
| `gpu_temp` | RO | u8 | GPU temperature °C |
| `fan0_rpm` | RO | u16 | Fan 1 RPM |
| `fan1_rpm` | RO | u16 | Fan 2 RPM |
| `power_profile` | RW | u8 | 0=battery, 1=balanced, 2=performance |

**Hardware access:**
- WMI BS (GUID `1F174999-3A4E-4311-900D-7BE7166D5055`)
- 8-byte input buffer, 80-byte output buffer
- Status word (bytes [0..1] LE) must equal 0 for success
- WMI methods: 0x02 (fans), 0x04 (CPU), 0x06 (GPU), 0x07 (profile)
- **No direct fan PWM control** — fans managed by power profile selection

### Unified DKMS Configuration

```
tux-kmod/
├── dkms.conf              # Unified: builds all 5 modules
├── Makefile               # Top-level: delegates to sub-module Makefiles
├── README.md
├── tuxedo-ec/
├── tuxedo-uw-fan/
├── tuxedo-tuxi/
├── tuxedo-clevo/
└── tuxedo-nb04/
```

**dkms.conf:**
```ini
PACKAGE_NAME="tux-kmod"
PACKAGE_VERSION="0.1.0"
AUTOINSTALL="yes"

BUILT_MODULE_NAME[0]="tuxedo_ec"
BUILT_MODULE_LOCATION[0]="tuxedo-ec/"
DEST_MODULE_LOCATION[0]="/extra"

BUILT_MODULE_NAME[1]="tuxedo_uw_fan"
BUILT_MODULE_LOCATION[1]="tuxedo-uw-fan/"
DEST_MODULE_LOCATION[1]="/extra"

# ... modules 2-4
```

### Justfile Recipes

```just
kmod-build       # Build all kernel modules
kmod-install     # dkms add + build + install
kmod-remove      # dkms remove
kmod-load-uw     # modprobe tuxedo_uw_fan (for testing)
kmod-unload-uw   # rmmod tuxedo_uw_fan
```

### Module Loading

- All 5 modules are independent — no loading order required
- Each probes its own hardware at load time; gracefully fails if hardware not present
- udev rules (optional): auto-load correct module based on DMI match
