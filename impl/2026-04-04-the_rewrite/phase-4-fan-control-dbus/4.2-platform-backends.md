### UniwillFanBackend

```rust
pub struct UniwillFanBackend {
    sysfs: SysfsReader,   // base: /sys/devices/platform/tuxedo-uw-fan
}

impl FanBackend for UniwillFanBackend {
    fn read_temp(&self) -> io::Result<u8> {
        self.sysfs.read_u8("cpu_temp")
    }

    fn write_pwm(&self, fan_index: u8, pwm: u8) -> io::Result<()> {
        // Convert PWM 0-255 → EC scale 0-200
        let ec_value = (pwm as u16 * 200 / 255) as u8;
        let attr = format!("fan{}_pwm", fan_index);
        self.sysfs.write_u8(&attr, ec_value)?;
        // Ensure manual mode
        self.sysfs.write_u8("fan_mode", 1)
    }

    fn set_auto(&self, _fan_index: u8) -> io::Result<()> {
        self.sysfs.write_u8("fan_mode", 0)
    }

    fn num_fans(&self) -> u8 { 2 }
    // ... read_pwm, read_fan_rpm
}
```

### Other Platform Backends

| Backend | Key Differences |
|---------|----------------|
| **TuxiFanBackend** | Native 0–255 PWM (no scale conversion). Temp in tenth-Kelvin → convert: `(val - 2730) / 10`. Has RPM. |
| **ClevoFanBackend** | Read fan_info u32, parse: duty=info&0xFF, temp=(info>>8)&0xFF, rpm=(info>>16)&0xFFFF. Write fan_speed as packed u32. |
| **Nb05FanBackend** | Uses `pread`/`pwrite` on binary `ec_ram` attribute. Two variants: ranges (Pulse 2-fan) and onereg (InfinityFlex 1-fan). |
| **Nb04** | No FanBackend — implements ProfileBackend only (power_profile RW). |

### Platform Initialization

```rust
pub fn init_fan_backend(device: &DetectedDevice) -> Option<Arc<dyn FanBackend>> {
    match device.descriptor.platform {
        Platform::Uniwill => Some(Arc::new(UniwillFanBackend::new()?)),
        Platform::Tuxi => Some(Arc::new(TuxiFanBackend::new()?)),
        Platform::Clevo => Some(Arc::new(ClevoFanBackend::new()?)),
        Platform::Nb05 => Some(Arc::new(Nb05FanBackend::new(device)?)),
        Platform::Nb04 => None,  // no direct fan control
    }
}
```
