# 5.2 — AC/Battery Auto-Switching

**Phase:** 5 — Profile Management
**Estimate:** 1 day (6 hours)
**Depends on:** 5.1 (profiles exist)
**Unlocks:** 5.3 (D-Bus exposes auto-switch config)

## Goal

Automatically switch profiles when the laptop transitions between AC power and battery. Event-driven via inotify on power supply sysfs — no polling.

### Power State Detection

```rust
pub enum PowerState {
    Ac,
    Battery,
}

pub fn detect_power_state() -> io::Result<PowerState> {
    // Read /sys/class/power_supply/AC*/online or /sys/class/power_supply/ADP*/online
    // "1" → Ac, "0" → Battery
}
```

### PowerStateMonitor

```rust
pub struct PowerStateMonitor {
    state_tx: watch::Sender<PowerState>,
}

impl PowerStateMonitor {
    pub fn new() -> (Self, watch::Receiver<PowerState>);

    /// Watch /sys/class/power_supply/ via inotify for changes.
    /// On change, re-detect power state and send to channel.
    pub async fn run(&self, shutdown: broadcast::Receiver<()>);
}
```

**Implementation:**
- Use `tokio::fs::watch` or `inotify` crate on `/sys/class/power_supply/*/online`
- On change event: re-read power state, send via `watch::Sender`
- Debounce: ignore rapid transitions within 500ms (power supply can fluctuate)

### Auto-Switch Integration

In `main.rs`:
```rust
// On power state change:
let mut power_rx = power_monitor.subscribe();
tokio::spawn(async move {
    while let Ok(()) = power_rx.changed().await {
        let state = *power_rx.borrow();
        let profile_id = match state {
            PowerState::Ac => &assignments.ac_profile,
            PowerState::Battery => &assignments.battery_profile,
        };
        if let Some(profile) = store.get(profile_id) {
            applier.apply(profile, &device).await?;
            tracing::info!("Switched to profile '{}' ({:?})", profile.name, state);
        }
    }
});
```
