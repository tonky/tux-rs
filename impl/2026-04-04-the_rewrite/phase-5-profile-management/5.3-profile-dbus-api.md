# 5.3 — Profile D-Bus API

**Phase:** 5 — Profile Management
**Estimate:** 1 day (6 hours)
**Depends on:** 5.1, 5.2, 4.3
**Unlocks:** Phase 6 (TUI uses this API)

## Goal

Expose profile CRUD, auto-switch configuration, and capability reporting via D-Bus. This completes the daemon's API — the TUI can now connect and manage everything.

## Deliverables

### D-Bus Interface Extensions

Add to existing `com.tuxedocomputers.tccd` service:

#### Profile Interface (`com.tuxedocomputers.tccd.Profile`)

| Method | Signature | Description |
|--------|-----------|-------------|
| `ListProfiles` | `() → String` | TOML array of all profiles (builtins + custom) |
| `GetProfile` | `(id: &str) → String` | Single profile as TOML |
| `CreateProfile` | `(toml: &str) → String` | Create, return new ID |
| `UpdateProfile` | `(id: &str, toml: &str)` | Update custom profile |
| `DeleteProfile` | `(id: &str)` | Delete custom profile |
| `CopyProfile` | `(id: &str) → String` | Copy profile, return new ID |
| `SetActiveProfile` | `(id: &str, state: &str)` | Set profile for "ac" or "battery" |
| `GetProfileAssignments` | `() → String` | TOML with ac_profile + battery_profile |

**Signals:**
- `ProfileChanged(profile_id: String, state: String)` — emitted on profile activation

#### Settings Interface (`com.tuxedocomputers.tccd.Settings`)

| Method | Signature | Description |
|--------|-----------|-------------|
| `GetGlobalSettings` | `() → String` | TOML global settings |
| `SetGlobalSettings` | `(toml: &str)` | Update global settings |
| `GetCapabilities` | `() → String` | TOML describing hardware capabilities |

#### System Interface (`com.tuxedocomputers.tccd.System`)

| Method | Signature | Description |
|--------|-----------|-------------|
| `GetSystemInfo` | `() → String` | TOML: daemon version, hostname, kernel |
| `GetPowerState` | `() → String` | "ac" or "battery" |

### Capabilities Response

```toml
[capabilities]
fan_control = true
fan_count = 2
keyboard_backlight = true
keyboard_type = "white"
charging_thresholds = false
charging_profiles = false
tdp_control = false
power_profiles = false
gpu_control = false
```

Derived from the detected `DeviceDescriptor.sensors`, `DeviceDescriptor.keyboard`, etc.

### Global Settings

```toml
[settings]
temperature_unit = "celsius"    # celsius | fahrenheit
fan_control_enabled = true
cpu_settings_enabled = true
```

### TOML Serialization Convention

All D-Bus methods that exchange configuration data use TOML strings. This keeps the D-Bus interface simple (strings only) while maintaining type safety through serialization.

```rust
// Server side
fn list_profiles(&self) -> fdo::Result<String> {
    let profiles = self.store.list();
    toml::to_string(&profiles).map_err(|e| fdo::Error::Failed(e.to_string()))
}

// Client side (TUI)
let toml_str = proxy.list_profiles().await?;
let profiles: Vec<TuxProfile> = toml::from_str(&toml_str)?;
```

### Error Handling

D-Bus errors for profile operations:
- `ProfileNotFound` — ID doesn't exist
- `ProfileReadOnly` — attempting to modify builtin
- `InvalidProfile` — TOML parse error or validation failure
- `StorageError` — filesystem permission or I/O error
