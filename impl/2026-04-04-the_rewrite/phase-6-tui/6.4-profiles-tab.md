# 6.4 — Profiles Tab

**Phase:** 6 — Terminal Interface
**Estimate:** 1 day (6 hours)
**Depends on:** 6.1, 6.3 (Form widget), 5.3 (Profile D-Bus API)

## Goal

Build the profiles management tab: list view with AC/battery markers, form-based profile editor, and all CRUD operations via D-Bus.

### List View

```
┌─ Profiles ────────────────────────────────────────────────────┐
│                                                                │
│  ● Max Energy Save          Default  ○ AC  ● BAT              │
│  ▸ Quiet                    Default  ○ AC  ○ BAT              │
│    Office                   Default  ● AC  ○ BAT              │
│    High Performance         Default  ○ AC  ○ BAT              │
│    My Gaming Profile        Custom   ○ AC  ○ BAT              │
│                                                                │
│  [Enter] Edit  [c] Copy  [d] Delete  [a] Set AC  [b] Set BAT │
└────────────────────────────────────────────────────────────────┘
```

**Markers:**
- `●` AC = this profile activates on AC power
- `●` BAT = this profile activates on battery
- `▸` = currently selected row
- Default vs Custom badge

**Keys (list mode):**
| Key | Action |
|-----|--------|
| `↑` / `↓` | Navigate profiles |
| `Enter` | Open profile editor |
| `c` | Copy selected profile (creates Custom copy) |
| `d` | Delete selected profile (Custom only) |
| `a` | Set selected as AC profile |
| `b` | Set selected as Battery profile |

### Profile State

```rust
pub struct ProfilesState {
    pub profiles: Vec<TuxProfile>,
    pub selected_index: usize,
    pub assignments: ProfileAssignments,
    pub mode: ProfilesMode,
}

pub enum ProfilesMode {
    List,
    Editor {
        form: Form,
        profile_id: String,
    },
}
```

### D-Bus Commands

```rust
enum Command {
    FetchProfiles,
    FetchProfileDetail(String),
    CopyProfile(String),
    DeleteProfile(String),
    SaveProfile { id: String, toml: String },
    SetActiveProfile { id: String, state: String },
    FetchAssignments,
}
```
