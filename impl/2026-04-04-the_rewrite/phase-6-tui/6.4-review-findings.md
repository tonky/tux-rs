# Phase 6.4 Review Findings — Profiles Tab

## Conformance Review
**Verdict: 95% CONFORMS**

All CRUD operations, UI rendering, D-Bus integration, and form-based editing implemented per spec. Minor deviation: `FetchProfileDetail` and `FetchAssignments` combined into single `FetchProfiles` command (optimization).

## Quality Review Findings

### CRITICAL — Acted Upon ✅

1. **Silent profile save failure when profile deleted while editor open**
   - **Issue:** Save handler silently did nothing if the profile was deleted externally.
   - **Fix:** Added error feedback (`"Profile no longer exists"`) and auto-return to list mode on save attempt.

2. **Profile deletable while editor is open (stale state)**
   - **Issue:** Editor could reference a profile that no longer exists in the list.
   - **Fix:** Added auto-close of editor in `ProfileList` event handler when the edited profile disappears from the updated list.

3. **No error feedback for profile-not-found on save**
   - **Issue:** Same as #1 — combined fix.

### IMPROVEMENTS — Acted Upon ✅

4. **Status messages never expire**
   - **Fix:** Status message is cleared on any list interaction (navigation, entering editor, etc.).

### IMPROVEMENTS — Postponed

5. **Per-frame profile lookup for editor title** — Minor perf concern; caching profile name in `ProfilesMode::Editor` adds coupling. The lookup is O(n) on a small list (<20 profiles). Not worth the complexity.

6. **String-based field mapping in `apply_form_to_profile`** — Valid concern about fragility. Would need enum-based field IDs. Deferring to Phase 8 polish if it causes maintenance issues.

7. **Unknown CPU governors silently mapped to default** — Acceptable behavior. The governor list covers standard Linux governors. Exotic values from the daemon would be unusual.

8. **No tab-switch protection while editing** — UX tradeoff. Blocking tab switches feels heavy; form state preserved until editor re-enter. Matches common TUI patterns (e.g., htop). Defer.

### MINOR — Acted Upon ✅

9. **Delete selected profile + selection clamping** — Added test `profiles_delete_selected_clamps_index`.
10. **Empty profile list edge case** — Added test `profiles_empty_list_then_populate`.

### MINOR — Ignored

11. **Form field index OOB debug assertion** — Form already guarantees `selected` is valid via `adjust()` clamping. Adding `debug_assert!` adds noise without value.
12. **Status message truncation on narrow terminals** — Minor UX edge case. Ratatui handles this gracefully by clipping. Not worth wrapping for a status line.
13. **No timeout for long-running D-Bus operations** — Valid but scope creep. The daemon is local; timeouts would add complexity. Defer to Phase 8.
