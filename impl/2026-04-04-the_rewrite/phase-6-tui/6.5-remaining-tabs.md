# 6.5 — Remaining Tabs (Settings, Keyboard, Charging, Power, Display, Webcam)

**Phase:** 6 — Terminal Interface
**Estimate:** 1 day (6 hours)
**Depends on:** 6.1, 6.3 (Form widget)
**Unlocks:** Phase 7 (features wire into these tabs)

## Goal

Implement the remaining 6 tabs. All use the generic Form widget — each tab is a form backed by a D-Bus get/set pair. This is primarily wiring work; the Form widget does the heavy lifting.

## Deliverables

### Module Structure

```
tux-tui/src/views/
├── settings.rs          # Global settings
├── keyboard.rs          # Keyboard backlight
├── charging.rs          # Charging thresholds
├── power.rs             # GPU/power info + settings
├── display.rs           # Display brightness/refresh
└── webcam.rs            # Webcam device controls
```

### Settings Tab

**Fields:**
- Temperature Unit: Select ["Celsius", "Fahrenheit"]
- Fan Control Enabled: Bool
- CPU Settings Enabled: Bool

**D-Bus:** `GetGlobalSettings` / `SetGlobalSettings`

### Keyboard Tab

**Fields:**
- Brightness: Number (0–100, step 5)
- Color: Text (hex "#rrggbb")
- Mode: Select ["Static", "Breathe", "Rainbow"]

**D-Bus:** `GetKeyboardState` / `SetKeyboardState`

**Note:** Fields disabled if device has `KeyboardType::None`. Mode options adapt to device capabilities.

### Charging Tab

**Fields:**
- Charging Profile: Select ["High Capacity", "Balanced", "Stationary"]
- Charging Priority: Select ["Charge", "Performance"]
- Start Threshold: Number (0–100%, step 5)
- End Threshold: Number (0–100%, step 5)

**D-Bus:** `GetChargingSettings` / `SetChargingSettings`

**Note:** Entire tab disabled if device has `ChargingCapability::None`. Shows "Not supported on this device."

### Power Tab

**Layout:** Info block (read-only) + settings form.

```

│                                                                │

│  │ NVIDIA RTX 4060         │ │ Intel Iris Xe            │     │
│  │ Temp: 45°C              │ │ Usage: 12%               │     │
│  │ Usage: 3%  Power: 15W   │ │                          │     │
│  └─────────────────────────┘ └──────────────────────────┘     │
│                                                                │
│  ── Settings ──────────────────────                           │
│  TGP Offset              [  0]                                │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

**D-Bus:** `GetGpuInfo` (info block), `GetPowerSettings` / `SetPowerSettings`

### Display Tab

**Fields:**
- Brightness: Number (0–100, step 5)

**D-Bus:** `GetDisplaySettings` / `SetDisplaySettings`

### Webcam Tab

**Special layout:** Device selector (multiple cameras) + per-device controls.

```
┌─ Webcam ──── [Camera 1] [Camera 2] ──────────────────────────┐
│                                                                │
│  Brightness          [  50]                                    │
│  Contrast            [  50]                                    │
│  Exposure            [  30]                                    │
│  Auto Exposure       [✓]                                       │
│                                                                │
│  [s] Save   [←][→] Switch device                              │
└────────────────────────────────────────────────────────────────┘
```

**D-Bus:** `ListWebcamDevices`, `GetWebcamControls(device)` / `SetWebcamControls(device, ...)`

### Capability-Aware Rendering

Each tab checks device capabilities from `GetCapabilities`:
```rust
fn render_tab(frame: &mut Frame, area: Rect, model: &Model) {
    if !model.capabilities.charging_thresholds {
        render_unsupported(frame, area, "Charging control not available on this device");
        return;
    }
    // ... render form
}
```

### D-Bus Client Methods

```rust
impl DaemonClient {
    // Settings
    pub async fn get_global_settings(&self) -> Result<String>;
    pub async fn set_global_settings(&self, toml: &str) -> Result<()>;

    // Power
    pub async fn get_gpu_info(&self) -> Result<String>;
    pub async fn get_power_settings(&self) -> Result<String>;

}
```
