# 7.1 — ITE Keyboard Backlight (HID)

**Phase:** 7 — Feature Parity
**Estimate:** 1 day (6 hours)
**Depends on:** 4.3 (D-Bus Keyboard interface)
**Unlocks:** Independent

## Goal

Port the ITE USB HID keyboard backlight control from tuxedo-drivers-rs. Four ITE chip families are supported, each with a different USB protocol. All operate in userspace via `/dev/hidraw*` — no kernel module needed.

## Deliverables

### Module Structure

```
tux-daemon/src/
├── hid/
│   ├── mod.rs             # KeyboardLed trait, Rgb struct, re-exports

│   ├── ite8297.rs         # RGB lightbar
│   ├── ite829x.rs         # Per-key 6×20 matrix
│   └── color_scaling.rs   # Per-model RGB corrections
└── ...
```

### KeyboardLed Trait

```rust
pub struct Rgb { pub r: u8, pub g: u8, pub b: u8 }

pub trait KeyboardLed: Send + Sync {
    fn set_brightness(&mut self, brightness: u8) -> io::Result<()>;
    fn set_color(&mut self, zone: u8, color: Rgb) -> io::Result<()>;
    fn set_mode(&mut self, mode: &str) -> io::Result<()>;
    fn zone_count(&self) -> u8;
    fn turn_off(&mut self) -> io::Result<()>;
    fn turn_on(&mut self) -> io::Result<()>;
    fn flush(&mut self) -> io::Result<()>;
    fn device_type(&self) -> &str;
    fn available_modes(&self) -> Vec<String>;
}
```

### ITE Chip Families

| Family | USB PID | LED Layout | Protocol |
|--------|---------|-----------|----------|
| ITE 8291 | 0x8291 | Per-key 6×21 (126 LEDs) + 4 zones | SET_FEATURE reports |
| ITE 8291 LB | 0x6010, 0x7000, 0x7001 | Lightbar (3 variants) | SET_FEATURE reports |
| ITE 8297 | 0x8297 | RGB lightbar | SET_FEATURE reports |
| ITE 829x | 0x8910 | Per-key 6×20 | SET_FEATURE reports |

### Color Scaling

Per-model RGB corrections to normalize brightness across different LED hardware:
```rust
pub struct ColorScaling {
    pub r_scale: f32,  // 0.0–1.0, e.g., 0.8 for dimmer red
    pub g_scale: f32,
    pub b_scale: f32,
}

pub fn scale_for_model(product_sku: &str) -> ColorScaling;
```
