# 7.2 — Charging Thresholds

**Phase:** 7 — Feature Parity
**Estimate:** 1 day (6 hours)
**Depends on:** 4.3 (D-Bus), Phase 3 (kernel shims)
**Unlocks:** Independent

## Goal

Implement charging threshold control for Clevo (ACPI flexicharger) and Uniwill (EC charge profile + priority). Allows users to limit charge to extend battery lifespan.

## Deliverables

### Module Structure

```
tux-daemon/src/
├── charging/

│   ├── clevo.rs           # Clevo flexicharger via sysfs
│   └── uniwill.rs         # Uniwill EC profile + priority
└── ...
```

### ChargingBackend Trait

```rust
pub trait ChargingBackend: Send + Sync {
    fn get_start_threshold(&self) -> io::Result<u8>;     // 0–100%
    fn set_start_threshold(&self, pct: u8) -> io::Result<()>;
    fn get_end_threshold(&self) -> io::Result<u8>;
    fn set_end_threshold(&self, pct: u8) -> io::Result<()>;
    fn get_profile(&self) -> io::Result<Option<String>>;
    fn set_profile(&self, profile: &str) -> io::Result<()>;
    fn get_priority(&self) -> io::Result<Option<String>>;
    fn set_priority(&self, priority: &str) -> io::Result<()>;
}
```

### Clevo Flexicharger

- Start threshold: battery level at which charging begins
- End threshold: battery level at which charging stops
- Sysfs: via existing Clevo ACPI platform device or `tuxedo_keyboard` compat

### Uniwill EC Charging

- Charge profiles: "high_capacity", "balanced", "stationary"
- Priority: "charge" (charge first) vs "performance" (TDP first)
- EC registers via `tuxedo-uw-fan` shim (extended attributes) or `tuxedo_keyboard` compat

### D-Bus Charging Interface

```rust
// com.tuxedocomputers.tccd.Charging
fn GetChargingSettings() -> String;    // TOML: profile, priority, start/end thresholds
fn SetChargingSettings(toml: &str);
```

### Profile Integration

Charging settings are part of `TuxProfile.charging` — automatically applied on profile switch.
