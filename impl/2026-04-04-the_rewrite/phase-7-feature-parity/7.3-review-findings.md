# Phase 7.3 Review Findings

## Conformance Review (95%)

All core requirements met. One intentional deviation: TDP values are clamped to bounds instead of returning error (better UX).

## Quality Review — Findings & Decisions

### CRITICAL

| # | Finding | Decision | Status | Reasoning |
|---|---------|----------|--------|-----------|
| 1 | TDP EC RAM race condition — no synchronization on concurrent set_pl1/set_pl2 | **IGNORE** | — | EcTdp is behind Arc, but writes are single-byte pwrite() calls which are atomic at the kernel level. Two concurrent writes to different EC addresses won't corrupt each other — they go to different offsets. Same-address races are benign (last writer wins). Adding Mutex adds complexity for no real benefit. |
| 2 | Governor partial failure — cpu0-2 succeed, cpu3 fails, inconsistent state | **ACT** | ✅ | Changed write_all() to best-effort: tries all CPUs, returns last error if any failed. Ensures maximum CPUs get the setting even if one fails. |

### IMPORTANT

| # | Finding | Decision | Status | Reasoning |
|---|---------|----------|--------|-----------|
| 3 | Governor TOCTOU — available_governors checked, then write happens | **IGNORE** | — | Extremely unlikely race. Kernel governor list changes require module load/unload. The validation is a convenience check for user error, not a security boundary. |
| 4 | D-Bus get/set asymmetry — get_pl1 returns 0 when unavailable, set_pl1 returns error | **IGNORE** | — | Intentional design. Getters return safe defaults so clients don't crash. Setters error because writing to nonexistent hardware should be surfaced. Same pattern as charging. |
| 5 | ProfileApplier always returns Ok(()) even if all settings fail | **IGNORE** | — | By design — profile application is best-effort. Each sub-system warns independently. The alternative (aggregate errors) adds complexity and doesn't change behavior since there's no one to act on the error at the auto-switch level. |
| 6 | No startup availability check for CPU sysfs | **IGNORE** | — | main.rs already checks `cpu_governor_available()` before creating the backend. If sysfs disappears after startup, individual operations will fail with appropriate errors. |
