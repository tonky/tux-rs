# 7.4 — Suspend/Resume + WMI Events

**Phase:** 7 — Feature Parity
**Estimate:** 1 day (6 hours)
**Depends on:** 4.4 (daemon lifecycle), 7.1 (keyboard state to save)
**Unlocks:** Independent

## Goal

Save and restore hardware state (fan mode, keyboard brightness/color) across suspend/resume cycles. Handle WMI keyboard hotkeys (e.g., Fn+brightness) for Clevo/Uniwill platforms.

## Deliverables

### Suspend/Resume

**Event source:** logind D-Bus `PrepareForSleep` signal

```rust
pub struct SleepHandler {
    fan_backend: Arc<dyn FanBackend>,
    keyboard: Option<Arc<Mutex<dyn KeyboardLed>>>,
    saved_state: Mutex<Option<SavedState>>,
}

struct SavedState {
    fan_mode: FanMode,
    fan_curve: Vec<FanCurvePoint>,
    keyboard_brightness: Option<u8>,
    keyboard_color: Option<Rgb>,
}

impl SleepHandler {
    /// Called on PrepareForSleep(true) — going to sleep
    pub async fn on_suspend(&self);

    /// Called on PrepareForSleep(false) — waking up
    pub async fn on_resume(&self);
}
```

**Suspend flow:**
1. Save current fan mode, active curve, keyboard state
2. Restore fans to auto (safe for sleep)
3. Turn off keyboard LEDs (save power)

**Resume flow:**
1. Wait 1–2 seconds (hardware initialization)
2. Restore fan mode and curve
3. Restore keyboard brightness and color
4. Re-discover ITE HID keyboards (USB may re-enumerate)

### WMI Event Handling

**Clevo hotkeys** (via WMI event GUID `ABBC0F6B`):
- Fn+brightness up/down
- Fn+fan mode toggle
- Other vendor hotkeys

**Uniwill hotkeys:**
- Input event devices from kernel shim
- Brightness key events

**Implementation:**
```rust
pub struct HotkeyHandler {
    fan_config_tx: watch::Sender<FanConfig>,
    keyboard: Option<Arc<Mutex<dyn KeyboardLed>>>,
}

impl HotkeyHandler {
    pub async fn on_hotkey(&self, key: HotkeyEvent);
}

pub enum HotkeyEvent {
    BrightnessUp,
    BrightnessDown,
    FanModeToggle,    // Cycle: Auto → Quiet → Performance → Auto
}
```

**Event sources:**
- `/dev/input/event*` devices — monitor with `evdev` crate or raw read
- Or: rely on existing desktop environment handling and only handle fan toggle

### Scope Limitation

For the first implementation:
- **Must have:** suspend/resume state save/restore
- **Nice to have:** WMI hotkey handling (can defer if complex)
- **Skip:** input event device monitoring if DE handles brightness keys
