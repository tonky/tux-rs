# 7.5 — NVIDIA GPU Power Control

**Phase:** 7 — Feature Parity
**Estimate:** 1 day (6 hours)
**Depends on:** 4.3 (D-Bus), Phase 3 (EC shim for NB02)
**Unlocks:** Independent

## Goal

Implement NVIDIA GPU power control for NB02 models (cTGP offset, dynamic boost) and GPU telemetry for the Power tab (temperature, usage, power draw).

## Deliverables

```
└── ...
```

### GPU Telemetry

```rust
pub struct GpuInfo {
    pub name: String,
    pub temperature: Option<f32>,    // °C
    pub usage_percent: Option<u8>,
    pub power_draw_w: Option<f32>,
    pub gpu_type: GpuType,
}

pub enum GpuType { Discrete, Integrated }

pub trait GpuBackend: Send + Sync {
    fn get_gpu_info(&self) -> io::Result<Vec<GpuInfo>>;
}
```

**Data sources (priority order):**
1. hwmon sysfs: `/sys/class/hwmon/hwmon*/` (nvidia driver exposes temp, power)
2. `nvidia-smi --query-gpu=...` fallback (if hwmon not available)
3. Intel iGPU: `/sys/class/drm/card*/device/` or i915 hwmon

### NB02 NVIDIA Power Control

NB02-specific EC registers for GPU TGP management:

```rust
pub struct Nb02NvidiaPower {
    ec: EcRam,
}

impl Nb02NvidiaPower {
    /// Get/set configurable TGP offset (cTGP)
    pub fn get_ctgp_offset(&self) -> io::Result<i32>;
    pub fn set_ctgp_offset(&self, offset: i32) -> io::Result<()>;

    /// Get/set dynamic boost enable
    pub fn get_dynamic_boost(&self) -> io::Result<bool>;
    pub fn set_dynamic_boost(&self, enabled: bool) -> io::Result<()>;
}
```

### D-Bus Power Interface

```rust
// com.tuxedocomputers.tccd.Power
```

### Scope

- **Primary:** GPU telemetry (temp, usage, power) for the Power/Dashboard tabs
- **Secondary:** NB02 cTGP control (narrow hardware scope — NB02 only)
- **Skip:** PRIME mode switching (handled by system tools), vfio passthrough
