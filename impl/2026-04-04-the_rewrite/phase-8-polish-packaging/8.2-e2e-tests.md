# 8.2 — End-to-End Tests

**Phase:** 8 — Polish & Packaging
**Depends on:** 8.1 (Integration Test Harness)
**Unlocks:** 8.5 (Missing Pieces)

## Goal

Build full-stack E2E tests that exercise core → daemon → D-Bus → TUI in a single test run. Tests validate per-device behavior driven by capabilities, without actual hardware. Establish shared D-Bus types in `tux-core` so daemon and TUI agree on wire formats.

## Deliverables

### Shared D-Bus Types in tux-core

Move D-Bus request/response structures into `tux-core` so both `tux-daemon` and `tux-tui` use the same types:

```rust
// tux-core/src/dbus_types.rs

/// Fan telemetry as returned by D-Bus GetFanInfo
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct FanInfoResponse {
    pub fans: Vec<FanData>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct FanData {
    pub rpm: u32,
    pub temp_celsius: f64,
    pub duty_percent: u8,
    pub max_rpm: u32,
}

/// Keyboard info as returned by D-Bus GetKeyboardInfo
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct KeyboardInfoResponse {
    pub keyboards: Vec<KeyboardData>,
}

/// Charging settings as returned by D-Bus
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ChargingSettingsResponse {
    pub profile: String,
    pub priority: String,
    pub start_threshold: u8,
    pub end_threshold: u8,
}

/// GPU info as returned by D-Bus GetGpuInfo
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct GpuInfoResponse {
    pub gpus: Vec<GpuData>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct GpuData {
    pub name: String,
    pub temp_celsius: Option<f64>,
    pub power_watts: Option<f64>,
    pub utilization_percent: Option<u8>,
}
```

### Device-Driven E2E Test Templates

Each device archetype gets a test that:
1. Creates a `MockSysfs` tree matching that device's hardware topology
2. Starts `TestDaemon` with a `DeviceDescriptor` matching the device
3. Exercises all capabilities the device supports via D-Bus
4. Verifies sysfs files reflect the expected state

```rust
/// Template for device-specific E2E tests.
/// Define a device by its capabilities and verify all features work.
fn run_device_e2e(descriptor: &DeviceDescriptor, sysfs: &MockSysfs) {
    let daemon = TestDaemon::start_with_descriptor(sysfs, descriptor).await;

    // Fan control — if device has fans
    if descriptor.num_fans > 0 {
        let fan_info = daemon.dbus_call("GetFanInfo", "").await;
        assert_eq!(fan_info.fans.len(), descriptor.num_fans as usize);

        daemon.dbus_call("SetFanCurve", test_curve_toml()).await;
        // Verify PWM files written
        for i in 0..descriptor.num_fans {
            assert!(sysfs.read_fan_pwm(i) > 0);
        }
    }

    // Keyboard — if device has keyboard backlight
    if descriptor.keyboard_type != KeyboardType::None {
        let kb_info = daemon.dbus_call("GetKeyboardInfo", "").await;
        assert!(!kb_info.keyboards.is_empty());
    }

    // Charging — if device supports it
    if descriptor.charging_thresholds {
        let settings = daemon.dbus_call("GetChargingSettings", "").await;
        assert!(settings.end_threshold > 0);
    }

    // GPU power — if device has it
    if descriptor.gpu_power {
        let gpu_info = daemon.dbus_call("GetGpuInfo", "").await;
        assert!(!gpu_info.gpus.is_empty());
    }

    daemon.stop().await;
}

    // Connect to daemon, execute command, return JSON result

```

E2E tests use this to verify TUI reads match daemon state:

```rust
#[tokio::test]
#[ignore]
async fn e2e_tui_reads_daemon_state() {
    let sysfs = MockSysfs::new();
    sysfs.add_fan("test", 2);
    let daemon = TestDaemon::start(&sysfs).await;

    // Set state via daemon D-Bus
    daemon.dbus_call("SetFanCurve", test_curve_toml()).await;

    // Read via TUI CLI
    let output = run_cli(CliCommand::DumpFanCurve, &daemon.client()).await;
    let curve: FanCurve = serde_json::from_str(&output).unwrap();
    assert_eq!(curve.points.len(), 5);

    daemon.stop().await;
}
```
