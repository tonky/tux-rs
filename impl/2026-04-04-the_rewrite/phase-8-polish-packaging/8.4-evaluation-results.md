## 2. Gaps Requiring Implementation (→ 8.5)

### Must-fix gaps

| Item | Priority | Effort | Notes |
|------|----------|--------|-------|
| **Fn Lock** (get/set/supported) | Medium | Small | Common on Uniwill laptops. Needs sysfs read/write for `/sys/devices/platform/tuxedo-tuxi/fn_lock` or similar |
| **Default profile values endpoint** | Low | Small | TCC GUI needs this for factory-reset profiles |
| **Display/Webcam daemon backends** | Low | Medium | TUI tabs exist but call non-existent daemon methods (`GetDisplaySettings`, `ListWebcamDevices`, etc.) |

## 3. Deliberately Omitted Features

| Feature | Reasoning |
|---------|-----------|
| **Aquaris water cooling** | Niche USB peripheral, extremely few users |
| **Tomte support tool** | External diagnostic tool, orthogonal to hardware control |
| **Sensor data collection** | Privacy concern; telemetry should be opt-in and separate |
| **System tray** | TUI runs in terminal; tray requires a separate GUI process |
| **Prime GPU switching** | Desktop-environment-specific; handled by `prime-select` or `supergfxctl` |
| **YUV420 output switch** | Display-server-specific, not portable |
| **GetIsX11** | Display-server detection not relevant for TUI |
| **ODM profiles/power limits** | OEM-specific modes, rarely used outside factory |
| **dGPU D0 metrics collection** | Telemetry; privacy concern |

### 4.2 Architecture Summary

```
Original: 27 C files, 9,689 LOC — monolithic kernel drivers owning state + policy
    tux-rs: 5 C shims, 1,582 LOC — stateless sysfs passthrough only

Key moves to userspace:
  • Keyboard LED control: 5 kernel drivers (2,448 LOC) → daemon HID (0 kernel LOC)
  • Fan control algorithms: kernel fan modules → daemon fan curve engine
  • EC access: ioctl multiplexer (1,031 LOC) → binary sysfs attr (292 LOC)
  • Platform detection: C compat check → Rust DMI device table
  • NB02/NB05 subsystems: 7 modules → daemon reads sysfs directly
```
