### Deliberately Omitted Features (with reasoning)

Document each omitted feature with explicit reasoning:
- **Aquaris water cooling** — Niche USB peripheral, very few users
- **Tomte support tool** — External diagnostic tool, orthogonal to hardware control
- **Sensor data collection** — Privacy concern; telemetry opt-in should be separate
- **System tray** — TUI runs in terminal; tray would require a separate GUI process
- **Prime GPU switching** — Desktop-environment-specific; handled by `prime-select` or `supergfxctl`
- **Webcam hardware toggle** — v4l2 control requires additional kernel interface not yet shimmed
- **YUV420 output switch** — Display server specific, not portable

### Kernel Driver Comparison

Compare `tuxedo-drivers/src/` (C) modules vs `tux-kmod/` shims:

```markdown
| C Module | LOC | tux-kmod Shim | LOC | What Changed |
|----------|-----|---------------|-----|-------------|
| clevo_acpi.c | ~800 | tuxedo-clevo/ | ~200 | Reduced to sysfs + EC passthrough |
| clevo_interfaces.c | ~400 | (merged) | — | Absorbed into clevo shim |
| uniwill_interfaces.c | ~600 | tuxedo-tuxi/ | ~150 | WMI wrapper → simple sysfs |
| uniwill_wmi.c | ~300 | (merged) | — | Absorbed into tuxi shim |
| tuxedo_io.c | ~500 | tuxedo-ec/ | ~100 | EC read/write binary attr only |
| tuxedo_nb04_... | ~2000 | tuxedo-nb04/ | ~100 | Minimal NB04 fan passthrough |
| uniwill_keyboard_driver.c | ~800 | — | — | Replaced by userspace HID (7.1) |
| clevo_keyboard.c | ~600 | — | — | Replaced by userspace HID (7.1) |
| tuxedo_keyboard_common.c | ~400 | — | — | Replaced by userspace HID (7.1) |
| ite_829x.c | ~300 | — | — | Replaced by userspace HID (7.1) |
| ite_8291.c | ~400 | — | — | Replaced by userspace HID (7.1) |
```

Key differences:
- Keyboard handling moved from kernel to userspace (HID via hidraw)
- Each shim is ~100-200 LOC vs ~400-800 LOC for C modules
- Fan control logic moved to userspace daemon (fan curve engine)
- EC access simplified to binary attribute (no ioctl multiplexing)
