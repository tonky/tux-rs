# 8.6 — TCC Compatibility Shim

**Phase:** 8 — Polish & Packaging
**Estimate:** 1 day (6 hours)
**Depends on:** 8.5 (Missing Pieces), 4.3 (D-Bus server), 5.3 (profile API), 7.1 (keyboard), 7.2 (charging)
**Unlocks:** Drop-in replacement for original TUXEDO Control Center

## Goal

Expose a backwards-compatible D-Bus interface that allows the original Angular/Electron TUXEDO Control Center to talk to `tux-daemon` without modification. The shim maps TCC's 58-method flat interface (JSON over D-Bus strings) to tux-rs internals.

## Background

The original TCC uses:
- **Bus name:** `com.tuxedocomputers.tccd`
- **Object path:** `/com/tuxedocomputers/tccd`
- **Interface:** `com.tuxedocomputers.tccd` (single flat interface)
- **Serialization:** JSON strings (D-Bus type `s`) for all complex data
- **Bus:** System bus
- **Signal:** `ModeReapplyPendingChanged(b)`

Since tux-daemon already owns the same bus name, the compat shim is an additional interface implementation on the same object path — coexisting with the native split interfaces (`*.Fan`, `*.Profile`, etc.).

### Translation Layer

Each method delegates to the tux-rs domain layer, then serializes to JSON matching the original TCC schema:

```rust
impl TccCompatInterface {
    fn get_fan_data_json(&self) -> String {
        let fans = self.daemon_state.fan_backend.read_all();
        let tcc_fan_data = IDBusFanData::from_domain(&fans);
        serde_json::to_string(&tcc_fan_data).unwrap_or_default()
    }

    fn get_profiles_json(&self) -> String {
        let profiles = self.daemon_state.profile_store.list();
        let tcc_profiles: Vec<ITccProfile> = profiles.iter()
            .map(ITccProfile::from_domain)
            .collect();
        serde_json::to_string(&tcc_profiles).unwrap_or_default()
    }
}
```

### TCC JSON Schema Types

Mirror the original TCC TypeScript interfaces as Rust structs with `serde::Serialize`:

```rust
/// Matches TCC's IDBusFanData
#[derive(Serialize)]
struct IDBusFanData {
    cpu: FanData,
    gpu1: FanData,
    gpu2: FanData,
}

/// Matches TCC's ITccProfile
#[derive(Serialize, Deserialize)]
struct ITccProfile {
    id: String,
    name: String,
    // ... fields matching TCC's schema exactly
}

/// Matches TCC's ITccSettings
#[derive(Serialize, Deserialize)]
struct ITccSettings {
    // ... fields matching TCC's schema exactly
}
```

### Feature Flag

The compat shim is behind a cargo feature flag so it can be excluded from builds:

```toml
[features]
default = ["tcc-compat"]
tcc-compat = ["serde_json"]
```

### Unsupported Methods

Methods for features not (yet) implemented in tux-rs return sensible defaults:
- `device_has_aquaris()` → `false`
- `get_display_modes_json()` → `"[]"`
- `get_is_x11()` → `-1` (unknown)
- `get_prime_state()` → `""`
- `get_is_unsupported_configurable_tgp_device()` → `false`
